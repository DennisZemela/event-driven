Datasets

Event-driven datasets contain the stream of events captured from a single or stereo event cameras. Each dataset can be played using the yarpdataplayer. Once the dataset is loaded in the yarpdataplayer the specified output port (default=/zynqGrabber/AE:o) is opened and can be connected to any event-driven processing module, the event-driven preprocessing module, or directly to the vFramerLite.

Each datasets may also contain, other sensors on the iCub (also re-playable with the yarpdataplayer) and ground truth measurements, given the task (e.g. ball locations, corner locations). Also included are Matlab scripts to generate the result figures as in the relevant papers.

Datasets recorded with the DVS and ATIS have the following properties which must be set correctly, in the module parameters and the cmake options, to interpret the data:

[DVS] width=128 height=128 VLIB_CODEC_TYPE=CODEC_128x128 VLIB_TIMER_BITS=24 VLIB_CLOCK_PERIOD_NS=128

[ATIS_20] width=304 height=240 VLIB_CODEC_TYPE=CODEC_304x240_20 VLIB_TIMER_BITS=24 VLIB_CLOCK_PERIOD_NS=80

[ATIS_24] width=304 height=240 VLIB_CODEC_TYPE=CODEC_304x240_24 VLIB_TIMER_BITS=30 VLIB_CLOCK_PERIOD_NS=80
